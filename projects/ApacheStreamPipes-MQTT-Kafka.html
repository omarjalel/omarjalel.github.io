Testing IIoT Gateway - Apache StreamPipes with Kafka Broker

Project description : vehicle simulator runs on a Raspberry Pi using a virtual CAN network, streaming telemetry data via MQTT through Apache StreamPipes and Kafka for real-time visualization and analysis

  • Run a vehicle simulator in a Raspberry Pi through virtual CAN Network
  • Expose some Data via Raspberry Pi VCAN (vehicle Speed, Engine Speed, Temperature...)
  • Run Apache StreamPipes Gateway and add MQTT Adapters for each telemetry
  • Deploy the adapters and read time series data from MQTT Broker
  • Forward data from Apache StreamPipes to Apache Kafka Broker hosted locally
  • Data Visualization
